### 2.4 Data model

New migration `supabase/migrations/043_listing_pages_slug_type.sql`:

```sql
ALTER TABLE listing_pages
  ADD COLUMN slug TEXT NOT NULL,
  ADD COLUMN city TEXT NOT NULL,
  ADD COLUMN page_type TEXT NOT NULL CHECK (page_type IN ('webpage', 'html')),
  ADD COLUMN style_id TEXT NOT NULL DEFAULT 'listing-page-v1';

CREATE UNIQUE INDEX listing_pages_slug_type_idx
  ON listing_pages (page_type, slug);
```

The `(page_type, slug)` uniqueness lets the same slug exist in both buckets (`/deals/12345-tacoma` and `/deals/html/12345-tacoma` are independent).

Existing columns (`is_active`, `address`, `price`, `html_content`, `inputs`, `lead_id`, `property_id`, `created_by`, `created_at`) all stay.

The `listing_pages` table is treated as a clean slate (no rows in production today). The migration adds these as `NOT NULL` without backfill since there's nothing to backfill.

### 2.5 Slug generation

New file `src/lib/listing-pages/slug.ts`:

1. Extract leading street number from `address` with `/^(\d+)/`. If no leading number, throw a clear error so the server action can return a useful failure to the client.
2. Slugify city: lowercase, spaces вҶ’ hyphens, strip non-alphanumerics. `"Federal Way"` вҶ’ `"federal-way"`.
3. Base slug: `${number}-${citySlug}` вҶ’ e.g., `"12345-federal-way"`.
4. Collision check: query existing slugs where `page_type = X` and `slug = base OR slug LIKE 'base-%'`. Find the max numeric suffix in use; if none, use `base`. Otherwise use `base-(N+1)`. So `12345-federal-way` вҶ’ `12345-federal-way-2` вҶ’ `-3` and so on.

City source: parsed from the typed `address` field on the form (not from `properties.city`), so the slug always reflects what the user actually entered.

### 2.7 Public routes

Both routes live at the **root domain** (not under `/app/`) and are public вҖ” no auth.

#### `/deals/[slug]/route.ts` вҖ” live webpage

A Next.js Route Handler (not a page component) вҖ” the saved HTML is a complete `<html>` document and we want it served raw, not embedded in Next's React tree:

```ts
export async function GET(_req, { params }) {
  const { slug } = await params
  const supabase = await createServerClient()
  const { data, error } = await supabase
    .from('listing_pages')
    .select('html_content')
    .eq('slug', slug)
    .eq('page_type', 'webpage')
    .eq('is_active', true)
    .single()

  if (error || !data) return new Response('Not found', { status: 404 })

  return new Response(data.html_content, {
    headers: { 'Content-Type': 'text/html; charset=utf-8' },
  })
}
```

Archived or deleted pages 404.
